Feedback Form Page

Technical Description

HTML Structure

  • The form tag is used to create the form, it encapsulates all the input fields and buttons.
  • The fieldset tag is used to group related elements together to improve the form's structure.
  • The legend tag provides a caption for the created group using fieldset tag.
  • The input tag with different type attributes(text, email, tel, radio, checkbox, file) is used to collect user inputs while using HTML validation.
  • The textarea tag allowes multi-line text input for user's to provide comments.
  • The select tag and option tag is used to create dropdowns and multi-select lists. datalist tag provides a searchable input field with predefined suggestions.
  • The button tag has type attributes such as submit and reset which is used to submit or reset the form.

CSS Styling

  • Background and Layout:
    • background-image: Used a background image overlayed with a gradient.
    • max-width and margin: The form is centered on the page with a maximum width of 600px for better readability.
    • border-radius and box-shadow: Used to round the corners and give a subtle shadow to the form.
  • Form Elements:
    • padding and border: Used for better visibility and interaction with the input fields.
    • hover and focus effects: To improve interactivity by making the color or style change of input fields and buttons when hovered over.
    • accent-color: Used to customize the color of checkboxes and radio buttons to match the website's theme.
  • File Upload:
    • Custom Button Styling: The file input is hidden, and a custom button is styled to trigger the file upload.
    • display: none: Hides the default file input, replacing it with a more visually appealing button.
  • Error Messages:
    • display: none and color: red: Error messages are made to be hidden by default and displayed in red when validation fails.
  • Responsive Design:
    • max-width and width: calc(100% - 20px): Used to ensure form elements scale properly on different screen sizes.
  • Hearts Rating:
    • flex Layout: Used to evenly space the hearts using display: flex and justify-content: space-between.
    • color and transition: Hearts change color on hover and when selected, providing visual feedback.

JavaScript Functionality

  • HTML5 Validation: Uses built-in validation (e.g. required, type="email", pattern) to ensure correct input formats.
  • Custom Error Messages: Displays error messages for invalid inputs (e.g. "Please enter a valid email address").
  • Hearts Rating: Changes the color of hearts based on user selection or hover, providing visual feedback.
  • Dynamically updates the character counter as the user types in the feedback textarea.
  • Form Submission:
    • addEventListener('submit'): Prevents the default form submission and collects form data using FormData.
    • window.open(): Opens a mailto link if the user chooses to send feedback via email.

Accessibility

  • Semantic HTML: Used form, fieldset, legend, and label tags to ensure screen readers can interpret the form structure and associate labels with inputs.
  • HTML5 Validation: Used required, type="email", and pattern attributes for real-time validation, with clear error messages for invalid inputs.
  • ARIA Attributes: Added ARIA roles and aria-describedby for error messages to improve screen reader compatibility and dynamic content accessibility.
  • Alt Text for Images: Provided descriptive alt text for all images (e.g., the loading GIF) to ensure visually impaired users understand their purpose.
  • Keyboard Navigation: Made all interactive elements (e.g., inputs, buttons, dropdowns) accessible via keyboard, with visible focus indicators for clarity.

Team Page

Technical Description

HTML Structure

  • Headings are used hierarchically (h1 for the page title, h2 for team member names, and h3 for roles) to improve accessibility and Search Engine Optimization.
  • The div tag is used to represent each team member by a card containing an image, name, role, and a hidden details section.
  • Hidden details about each team member’s tasks are revealed on hover, using CSS transitions.
  • The main tag encapsulates the main content of the team page, ensuring a clear structure for screen readers.

CSS Styling

  • Layout and Grid:
    • display: grid: The team members are arranged in a grid layout using CSS Grid, with a diamond-shaped positioning (grid-column and grid-row properties).
    • gap: 20px: Adds spacing between team member cards for better readability and visual appeal.
  • Hover Effects:
    • On hover, images scale up slightly (transform: scale(1.1)) and reduce opacity (opacity: 0.8), providing visual feedback.
    • Hidden details (e.g. tasks) become visible on hover with a fade-in effect (opacity: 1 and visibility: visible).
  • Used text-shadow to add a soft shadow to the page title (h1) for emphasis and visual depth.
  • Added padding-bottom to the main content to ensure the footer doesn’t overlap with the team member cards.
  • Typography:
    • Team member images are styled as circles using border-radius: 50% property.
    • Added subtle shadows to images and text elements using box-shadow property.
    • Used a semi-transparent background along with contrasting text color for details section to improve readability.

Accessibility

  • All images have descriptive alt text, ensuring screen readers can convey the content to visually impaired users.
  • Added to each team member card to make them focusable via keyboard navigation, improving accessibility for users who rely on keyboards or screen readers.
  • Semantic HTML: Used header, main, and footer for clear structure, and h1 to h3 for hierarchical headings, improving screen reader navigation.
  • Used high-contrast colors (e.g. dark text on light backgrounds) and clear typography to improve readability.

Content Page

Technical Description

HTML Structure

  • Each subtopic is wrapped in a div tag with a unique id for internal linking.
  • Lists (ul and ol) are used to present key points and facts in a structured manner.
  • Used a background video with an overlay text(div class="overlay-text") to introduce the theme of the page.
  • Added a fixed side navigation bar(div class="side-nav") which allows users to jump to different sections of the page using anchor links.
  • Added a fixed "Go to Top" button allows users to quickly return to the top of the page.

CSS Styling

  • Page Layout and Image Grid:
    • display: flex and grid: Used for organizing images in a grid layout and centering content.
    • position: fixed: The side navigation and "Go to Top" button are fixed to ensure they remain visible as users scroll.
  • Content Section Styling:
    • background-color and border-left: Added uniques colored backgrounds and left borders to the subtopic sections for visual distinction.
    • box-shadow and border-radius: To add depth and rounded corners to content sections.
  • Hover Effects:
    • transition and transform: To move the side navigation links slightly to the right on hover, providing visual feedback.
    • text-decoration: underline: Links are underlined on hover to indicate interactivity.
  • text-shadow: Used to add emphasis to the page title (h1) with a soft shadow.
  • max-width and margin: 0 auto: Used to center the content to a maximum width for better readability on larger screens.
  • object-fit: cover: Ensures images and videos maintain their aspect ratio and fill their containers.
  • z-index: Used to control the layering of elements (e.g. video overlay text appears above the video).

Accessibility

  • Semantic HTML: Used header, main, and footer for clear page structure, and h1 to h3 for hierarchical headings, improving screen reader navigation.
  • All images include a descriptive alt text (e.g. "Carbon Footprint Factors") to ensure screen readers can convey their purpose.
  • Interactive elements with hovering effects so that the users can see which element is currently focused.
  • Used high-contrast colours and clear typography to improve readability for users with visual impairments.
  • The page layout adapts to different screen sizes.

Challenges and Lessons Learned

Challenges and Resolutions:

1. Feedback Form Page

  • Implementing real-time validation for fields like email, phone number, and character limits was challenging, especially ensuring that error messages were clear and accessible.
    • Used HTML5 validation attributes (required, type="email", pattern) and custom JavaScript to provide real-time feedback. Error messages were styled with CSS to ensure visibility.
  • The hearts rating system was challenging as it required careful planning to be both functional and visually appealing.
    • Used radio buttons styled with CSS to create the hearts rating system. Added tabindex="0" and ARIA attributes to ensure keyboard accessibility.
  • Matching the style of file upload input with the website's theme while maintaining accessibility was harder than expected.
    • Hid the default file input and replaced it with a custom-styled button (label). Added visual feedback for uploaded files.

2. Team Page

  • Arranging team member cards in a diamond-shaped grid while ensuring responsiveness was complex.
    • Used CSS Grid with grid-column and grid-row properties to position team member cards in a diamond shape. Adjusted spacing with gap and margin.
  • Implementing hover effects that revealed additional details without disrupting the layout.
    • Used opacity and visibility transitions to smoothly reveal details on hover. Added tabindex="0" to make cards focusable and accessible via keyboard.

3. Content Page

  • Ensuring the video background was responsive and did not interfere with readability of overlay text required careful CSS adjustments.
    • Used object-fit: cover and z-index to ensure the video scaled properly and stayed behind the overlay text. Added a semi-transparent background to the text for better contrast.
  • Positioning the side navigation bar to remain fixed while scrolling without overlapping content was difficult.
    • Used position: fixed and z-index to keep the navigation bar visible and prevent overlap with content. Added padding to the main content to avoid overlap.
  • Creating a responsive image grid that maintained alignment and spacing across different screen sizes was challenging.
    • Used display: flex and flex-wrap to create a responsive grid. Adjusted margins and padding to ensure consistent spacing.

Lessons Learned:

  • Always prioritize accessibility to ensure inclusivity for all users.
  • Provide real-time validation and clear error messages for better usability.
  • Add interactive features without compromising functionality or accessibility.
  • Use tools such as CSS Grid & Flexbox for creating complex, responsive layouts efficiently.
  • Use contrast and spacing to guide users' attention to key content.
  • Use proper tags for better structure, Search Engine Optimization, and screen reader compatibility.

Compliance

The Janet Web Accessibility Guidelines ensure that web pages are accessible, usable, and compliant with legal standards. The Feedback Form, Content Page, and Team Page comply with these regulations as shown below.

Feedback Form Page:

  • Accessible Forms: Labels, error messages linked with ARIA attributes, and keyboard navigation ensure usability.
  • Validation: HTML5 validation and JavaScript real-time feedback improve user experience.
  • Contrast:
    • Text and background colors meet Web Content Accessibility Guidelines (WCAG) contrast ratios.
    • Clearly visible focus states for interactive elements.

Team Page

  • Keyboard Accessibility:
    • Team member cards are focusable.
    • Hover effects are accessible via keyboard.
    • Hidden details are revealed with ARIA attributes for screen readers.
  • Alt Text: Team member images include descriptive alt text for screen readers.
  • Responsive Layout: Diamond grid layout is responsive and adjusts for smaller screens.

Content Page

  • Semantic Structure:
    • Proper use of header, main, and footer tags.
    • Hierarchical use of h1 to h3 heading tags for better navigation and Search Engine Optimization.
  • Multimedia:
    • Images have descriptive alt text and background video includes an overlay text with suffiecient contrast for readability.
    • The side navigation is keyboard accessible, with focus states for links.
  • Responsive Design: Adapts to different screen sizes without overlapping, ensuring usability on mobile and desktop devices.
