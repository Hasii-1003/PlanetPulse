Template

Technical Description:

  1. Header Section - header
    • Includes a logo inside a div along with a title and subtitle inside a div
  2. Navigation Bar (nav Equivalent)
    • The navigation bar is implemented inside a div class="navbar", which contains an unordered list of links.
    • The target="_blank" attribute ensures that links open in a new tab.
  3. Main Content
    • A large h1 centered on the page provides a welcome message.
  4. Footer Section
    • The footer contains:
      • Page Editors Links – Placeholder links to different page editors.
      • Styling & Policy Links – Links to external CSS and policy pages.
      • Copyright Information – Displays ownership details.

CSS Techniques

  1. Banner and Navigation Bar
    • width: 100%; height: 65px; background-color: rgb(31, 103, 101);
      • The navigation bar stretches across the screen with a greenish background.
    • .navbar ul li
      • Displays menu items inline using display: inline-block;
      • The position: relative; allows for the hover effect.
  2. Logo and Title Styling.
    • .logo
      • The logo is positioned absolutely at the top-left.
    • .title-container
      • Uses display: flex; justify-content: center; to align elements properly.
    • .subtitle
      • Uses position: absolute; to place text below the title.
  3. Hover Effects (Navbar & Footer)
    • The navbar has an underline effect
  4. Footer Styling
    • .footer-content and .styling
      • Uses background-color for contrast.
      • text-align: center; for better alignment.
  5. Media Queries (Responsive Design)
    • Adjust font sizes for smaller screens.

Homepage

Technical Description:

  • The img tag is used throughout the page to display the logo, content images, and section visuals.
  • CSS styles are applied to control the size, border-radius, and placement of images.
  • Buttons for "Read More" use a> tags styled with .blog-btn to look like buttons.

CSS Techniques:

  • The .about-us, .mission, and .contents-container sections are designed with flexbox to center content and align elements properly.
  • The .contents-container uses display: flex; to arrange the content cards horizontally.
  • max-width, margin: auto, and text-align: center are used to make elements responsive.
  • The backdrop-filter: blur(10px); effect adds a glassmorphism look to .about-us.
  • The .about-us:hover effect scales the section slightly when hovered.
  • .blog-btn:hover changes the background color and text color for interaction feedback.
  • .card-contents:hover changes the border color.
  • Shadows: Used in .about-us, .card-contents, and .blog-btn to create depth.
  • Borders: Rounded corners are added to elements.
  • A consistent teal, blue, and white color scheme is maintained throughout the website.

Accessibility

  • Alt Text for Images with descriptions
  • Semantic HTML (header, nav, section, footer)
  • Proper Heading Order.
  • Buttons with Hover Effects
  • Use Contrast Ratios

Table

Technical Description:

  • table: Displays structured data.
  • th and td: Used for column headers and table data
  • Uses JavaScript (document.querySelectorAll('.expandable-row')) to toggle visibility of hidden tr elements.
  • Clicking on a row expands/collapses additional content.
  • Unicode Emojis are used to make the table visually engaging.
  • span class="icon">✔span: Represents status indicators.

CSS Techniques:

  • width: 100%; border-collapse: collapse;: Ensures the table spans the full width and removes extra spacing between borders.
  • box-shadow: 0 4px 8px rgb(1, 55, 17);: Adds depth with a shadow effect.
  • background-color: rgb(140, 208, 160);: Sets a background color.
  • th { position: sticky; top: 0; }: Keeps table headers visible during scrolling.
  • tr:hover { background-color: #f2f6bb; transition: background-color 0.3s ease; }: Smooth hover effect.
  • .progress-bar:
    • Uses a div inside another div to represent the progress.
    • Background is white (background-color: white;).
    • Inner bar changes width dynamically (style="width: 75%;").
  • Expandable Table Rows
    • .expandable-content { display: none; }: Initially hidden.
    • JavaScript toggles display: table-row; when a user clicks an expandable row
  • Interactive Tooltip
    • CSS Classes: .tooltip and .tooltiptext
    • Hidden by default (visibility: hidden;)
    • Becomes visible on hover (visibility: visible; opacity: 1;).
    • Positioned above the hovered element

Accessibility

  • aria-label="Climate Action Dashboard" provides better navigation for assistive technologies.
  • Sticky Table Headers (position: sticky; top: 0;): Keeps column headers visible when scrolling.
  • Alternating row colors (tr:nth-child(even)): Improves readability.
  • Hover effect (tr:hover): Highlights rows when hovered.

Content Page 02

Technical Description

  1. Title & Text Section
    • The h3 tag is used to display the title in the center
    • The title font is large and colored to stand out.
  2. Image Elements
    • img tags display images with specific sizes and styles.
    • Each image is wrapped in a div or figure to provide consistent spacing.
    • Hover effects (:hover) are added using CSS for interactive visuals.
  3. Image Slider
    • Uses a container div to hold multiple img elements.
    • CSS animations (@keyframes slide) create a smooth transition effect.
    • overflow: hidden; ensures only the current image is visible.
  4. Video Background
    • Uses the video tag with attributes like autoplay, muted, and loop
    • A separate div is positioned over the video to display text overlays.
    • The object-fit: cover; property ensures the video covers the entire section without distortion.
  5. Play/Pause Video Functionality
    • Used JavaScript to check if the video is playing or paused.
    • A button allows users to toggle playback with one click.

CSS Techniques:

  • color: black; or a dark shade ensures high contrast for readability
  • padding and margin are used to create proper spacing between elements.
  • border: 3px solid #color; adds a frame around images.
  • transition: 0.3s ease-in-out; ensures a smooth hover effect
  • On hover (:hover), the image changes color, rotates, or enlarges slightly.
  • The slider container has position: relative; overflow: hidden; to control image flow.
  • @keyframes are used to animate the images, making them slide automatically.
  • The video is positioned absolute inside a container to act as a background
  • The text overlay is styled using position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%);.
  • z-index ensures the text appears above the video

Accessibility

  • Screen Reader-Friendly (Semantic HTML, alt text, labels)
  • Keyboard Accessible (Tab navigation, focus styles)
  • High Contrast & Readability (Dark mode, scalable fonts)
  • Responsive for All Devices (Works on mobile, desktops, screen magnifiers)
  • Motion Sensitivity Considerations (No forced animations)
  • Accessible Multimedia (Captions, transcripts, proper controls)

  3. Ensuring consistent font sizes, colors, and spacing across all pages.
  4. Making the content engaging and easy to read.

Compliance

While creating my website, I made sure it followed Janet regulations by focusing on legal compliance, security, and appropriate content use.To comply with data protection laws like GDPR, I included a Privacy Policy page explaining how user data is collected and used. Even though my site doesn’t collect much data, I still mentioned things like cookies to ensure transparency. I also created a Terms of Use page to outline user responsibilities and prevent misuse.

I also followed Janet’s Acceptable Use Policy by keeping my content truthful, ethical, and free from copyrighted material. I made sure all images and text were either my own or royalty-free. Additionally, I considered basic accessibility by using clear headings and readable fonts, though I could improve this by adding alt text for images.
