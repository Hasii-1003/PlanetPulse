User Profile

Technical Description

The webpage is built using a combination of HTML for structure and CSS for styling, with a focus on creating a clean, modern, and interactive user experience.
Here's a breakdown of the key techniques and design choices:

  • HTML Structure: The page is structured with semantic HTML elements such as <header>, <section>, <div>, and <footer> to ensure clarity and accessibility.
  • CSS Styling: The page uses CSS for styling, including a gradient background, hover effects, and transitions to enhance interactivity. Flexbox and Grid layouts are used for responsive design.
  • Interactive Elements: JavaScript is used to dynamically update the rating sliders and their displayed values, providing real-time feedback to users.
  • Responsive Design: The layout is designed to be responsive, ensuring it adapts to different screen sizes and devices.
  • Calendar Grid: A 7-column grid is used to display the calendar, with hover effects and event markers for better user engagement.

Accessibility

Accessibility was addressed in the following ways:

  • Semantic HTML: Proper use of HTML5 elements like <header>, <main>, and <footer> ensures screen readers can navigate the page effectively.
  • ARIA Labels: Interactive elements like sliders and buttons include ARIA attributes to improve accessibility for users with disabilities.
  • Color Contrast: The color scheme was chosen to ensure sufficient contrast between text and background, making it easier to read for users with visual impairments.
  • Keyboard Navigation: All interactive elements are accessible via keyboard, ensuring users who cannot use a mouse can still navigate the page.

Sitemap

Technical Description

The sitemap page is implemented using a combination of HTML for structure and CSS for styling, with an interactive SVG graphic to visually represent the website's structure. Below are the key techniques and design choices:

  • HTML Structure: The page uses semantic HTML elements like <section>, <div>, and <svg> to create a clear and organized layout. The SVG graphic is embedded directly into the HTML, allowing for dynamic and interactive elements.
  • CSS Styling: CSS is used to style the page, including a gradient background, hover effects, and transitions for interactivity. Flexbox is used to center the SVG graphic and ensure responsiveness.
  • SVG Graphic: The sitemap is created using SVG elements like <circle> for nodes and <line> for connections. Each node represents a page on the website, and lines indicate the relationships between pages.
  • Interactivity: JavaScript is used to add tooltips that appear when hovering over nodes, providing additional information about each page. This enhances user experience and accessibility.
  • Responsive Design: The page is designed to be responsive, with the SVG graphic scaling appropriately to fit different screen sizes.

Accessibility

Accessibility was addressed in the following ways:

  • Semantic HTML: Proper use of HTML5 elements ensures screen readers can navigate the page effectively.
  • ARIA Attributes: Interactive elements like nodes and links include ARIA attributes to improve accessibility for users with disabilities.
  • Keyboard Navigation: All interactive elements are accessible via keyboard, ensuring users who cannot use a mouse can still navigate the page.
  • Tooltips: Tooltips provide additional context for each node, making the sitemap easier to understand for all users.

Content Page

Technical Description

The content page is designed to provide an engaging and informative experience using a combination of HTML for structure and CSS for styling. Below are the key techniques and design choices:

  • HTML Structure: The page uses semantic HTML elements like <header>, <nav>, <section>, and <footer> to create a clear and organized layout. Each section is logically divided to present information in a structured manner.
  • CSS Styling: CSS is used to style the page, including a gradient background, hover effects, and transitions for interactivity. Flexbox is used for layout management, ensuring responsiveness and alignment of elements.
  • Navigation Bar: The navigation bar is sticky, ensuring it remains visible as the user scrolls. It uses flexbox for alignment and horizontal scrolling for smaller screens, improving usability.
  • Subtopic Sections: Each subtopic section uses a flexbox layout to align images and content side by side. Hover effects and transitions are added to enhance interactivity and visual appeal.
  • Responsive Design: Media queries are used to ensure the page adapts to different screen sizes. On smaller screens, the layout switches to a vertical stack, and font sizes are adjusted for readability.
  • Go-to-Top Button: A fixed-position button is added to allow users to quickly scroll back to the top of the page. JavaScript is used to toggle its visibility based on scroll position.

Accessibility

Accessibility was addressed in the following ways:

  • Semantic HTML: Proper use of HTML5 elements ensures screen readers can navigate the page effectively.
  • ARIA Attributes: Interactive elements like the navigation bar and buttons include ARIA attributes to improve accessibility for users with disabilities.
  • Keyboard Navigation: All interactive elements are accessible via keyboard, ensuring users who cannot use a mouse can still navigate the page.
  • Contrast and Readability: The color scheme was chosen to ensure sufficient contrast between text and background, making it easier to read for users with visual impairments.
  • Responsive Design: The page is designed to be fully responsive, ensuring it is usable on a wide range of devices, including mobile phones and tablets.

Challenges and Lessons Learned

While working on this project, I encountered several challenges that tested my problem-solving skills and taught me valuable lessons. Below, I outline the key challenges I faced and how I resolved them:

  • Matching CSS Classes with the Global Template CSS:
    • Challenge: One of the initial challenges was ensuring that my custom CSS classes aligned with the global template's styling. There were conflicts between my styles and the predefined styles in the global template, which caused inconsistencies in the design.
    • Solution: To resolve this, I carefully reviewed the global template's CSS file and adjusted my custom classes to avoid naming conflicts. I also used more specific CSS selectors to ensure my styles took precedence where needed. Additionally, I utilized browser developer tools to debug and test the changes in real-time.
    • Lesson Learned: I learned the importance of understanding global styles and how to structure my CSS to avoid conflicts. This experience also improved my debugging skills using browser tools.
  • Learning SVG for the Sitemap:
    • Challenge: Creating the interactive SVG sitemap was a significant challenge, especially ensuring proper syntax and alignment. I struggled with correctly placing the nodes and lines, and debugging issues with missing or misplaced parentheses in the SVG code.
    • Solution: I resolved this by breaking down the SVG code into smaller, manageable sections and testing each part individually. I also used online SVG validators to check for syntax errors and ensure proper alignment. Additionally, I referred to documentation and tutorials to better understand SVG properties and attributes.
    • Lesson Learned: This experience taught me the importance of patience and attention to detail when working with complex code like SVG. I also gained a deeper understanding of SVG syntax and its capabilities.
  • Ensuring Responsiveness Across Devices:
    • Challenge: Making the page fully responsive for different screen sizes, especially mobile devices, was challenging. Some elements did not align properly or became too small to read on smaller screens.
    • Solution: I used media queries to adjust the layout, font sizes, and spacing for different screen sizes. I also tested the page on multiple devices and browsers to ensure consistency.
    • Lesson Learned: I learned the importance of designing with responsiveness in mind from the start and how to use media queries effectively to create a seamless user experience across devices.
  • Implementing Accessibility Features:
    • Challenge: Ensuring the page was accessible to all users, including those with disabilities, was a new challenge for me. I had to learn about ARIA attributes, semantic HTML, and keyboard navigation.
    • Solution: I incorporated ARIA attributes for interactive elements, used semantic HTML tags, and ensured all functionality was accessible via keyboard. I also tested the page with screen readers to identify and fix accessibility issues.
    • Lesson Learned: This experience taught me the importance of accessibility in web design and how to implement best practices to make websites inclusive for all users.

Overall, these challenges helped me grow as a developer and taught me valuable skills in debugging, problem-solving, and creating user-friendly, accessible, and responsive web pages.

Compliance

The pages I have created are designed to comply with the Janet Web Accessibility Regulations, which are based on the Web Content Accessibility Guidelines (WCAG) 2.1. These regulations ensure that web content is accessible to all users, including those with disabilities. Below, I outline how my pages adhere to these standards:

  • Semantic HTML:
    • I used semantic HTML elements such as <header>, <nav>, <main>, <section>, and <footer> to structure the content. This ensures that screen readers and other assistive technologies can interpret and navigate the page effectively.
  • Keyboard Accessibility:
    • All interactive elements, such as navigation links, buttons, and form inputs, are fully accessible via keyboard. Users can navigate the page using the Tab key, and focus states are clearly visible to indicate the currently selected element.
  • ARIA Attributes:
    • I incorporated ARIA (Accessible Rich Internet Applications) attributes to improve accessibility for dynamic content and interactive elements. For example, I used aria-label to provide descriptive text for icons and buttons, and role to define the purpose of certain elements.
  • Color Contrast and Readability:
    • I ensured sufficient color contrast between text and background colors to meet WCAG 2.1 AA standards. This makes the content readable for users with visual impairments or color blindness. I used tools like the WebAIM Contrast Checker to verify compliance.
  • Responsive Design:
    • The pages are fully responsive and adapt to different screen sizes, including mobile devices and tablets. This ensures that users can access the content on any device without losing functionality or readability.
  • Alt Text for Images:
    • All images include descriptive alt text to ensure that users with visual impairments can understand the content of the images through screen readers.
  • Form Accessibility:
    • Forms are designed with accessible labels and error messages. Each input field is associated with a <label> element, and error messages are clearly displayed to assist users in correcting mistakes.
  • Testing and Validation:
    • I used tools like the W3C Markup Validation Service and accessibility checkers (e.g., Lighthouse, Axe) to test the pages for compliance. While I aimed for full compliance, I acknowledge that some areas may require further improvement, such as ensuring all dynamic content is fully accessible.

While I have made significant efforts to ensure compliance with Janet regulations, I recognize that accessibility is an ongoing process. I am committed to continuously improving the accessibility of my pages and addressing any issues identified through user feedback or further testing.
